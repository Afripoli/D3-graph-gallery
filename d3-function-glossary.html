A collection of d3 functions




D3 is a collection of modules that can be loaded independently. Each module offers a set of functions. This document provides a list of them, with a short human readable explanation and some code examples.

d3.cluster()

d3.cluster() is used to create a cluster layout = to provide some coordinates to every member of a hierarchy

Calling d3.cluster() creates a function.

const clusterCreator = d3.cluster()

The new clusterCreator accepts a node object as input. See d3.hierarchy to build it. Process looks like:

// An object describing a hierarchy
const data = {
  name: "Boss",
  children: [
    {
      name: "ProductManager1"
    },
    {
      name: "ProductManager2",
      children: [
        {
          name: "Engineer1"
        },
        {
          name: "Engineer2"
        }
      ]
    }
  ]
};

// build a "node" object from it:
const node = d3.hierarchy(data);

clusterCreator is gonna provide some coordinates to all the nodes of the node object:

const nodeWithCoordinates = clusterCreator(node)

nodeWithCoordinates looks like:

{
  x: 0.45,
  y: 0,
  children: [
    {
      x: 0.2,
      y: 1,
      children: [Array]
    },
    {
      x: 1
      y: 0.8,
      children: [Array]
    }
  ]
}

It's now easy to use this object to create a viz!

d3.hierarchy()

d3.hierarchy() takes a hiearchic dataset and builds a node object from it.

What is a hiearchic dataset? It's a dataset that looks like this:

const data = {
  name: "Boss",
  children: [
    {
      name: "ProductManager1"
    },
    {
      name: "ProductManager2",
      children: [
        {
          name: "Engineer1"
        },
        {
          name: "Engineer2"
        }
      ]
    }
  ]
};

Basically, we have a boss in the company. Under him, 2 product managers. The second, ProductManager2, has 2 engineers working from him.

d3.hierarchy() is gonna read this format and build a node with it: an object that provides many interesting features and will be used as input for many other d3 functions.

const node = d3.hierarchy(data);

The node object looks pretty much like this:

{
  data: Object // an object representing the hierarchy
  height: 2 // # of levels of hierarchy
  depth: 0 // level of this node in the hierarchy. 0 = root node, the highest
  parent: null // no parent since this node is the root node
  children: Array[2] // an array of nodes that have this format as well
}

The node object also has several methods allowing to work with it.

  • node.leaves() returns an array with each leaf.
